

## Проблема

В системе **84 платежа** застряли в статусе `pending` (с августа 2025). Нет механизма автоматической очистки: если пользователь закрыл окно оплаты или платеж отклонен без вебхука, запись остается в pending навечно.

## Решение

### 1. Создать Edge-функцию `cleanup-stale-payments`

Функция будет:
- Находить все платежи со статусом `pending`, созданные более 1 часа назад
- Обновлять их статус на `expired`
- Логировать количество очищенных записей

```typescript
// Логика:
UPDATE payments SET status = 'expired', updated_at = now()
WHERE status = 'pending' AND created_at < now() - interval '1 hour'
```

### 2. Настроить pg_cron для автозапуска

Запускать функцию каждые 30 минут через `cron.schedule`.

### 3. Одноразовая очистка существующих 84 записей

Миграция для перевода всех текущих старых pending-платежей в `expired`.

### 4. Обновить RLS-политику `secure_payments_webhook_update`

Добавить `expired` в список допустимых статусов (`WITH CHECK`), чтобы service role мог выставлять этот статус.

**Файлы:**
- Новый: `supabase/functions/cleanup-stale-payments/index.ts`
- Миграция: одноразовая очистка + pg_cron расписание + обновление RLS
