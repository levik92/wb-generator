# План: квиз-лендинг `/quiz`

## Что строим

Новая рекламная страница `/quiz` — полный мини-лендинг + квиз в поп-апе с ветвлением по аудитории, минутным видео и переходом на страницу благодарности → регистрацию.

## Архитектура

```
/quiz                  → лендинг + кнопка «Хочу попробовать»
   └─ ResponsiveDialog → квиз (вопрос 1 → вопрос 2 → видео → CTA)
/quiz/thanks           → благодарность + автопереход на /auth?tab=register (10 сек)
```

Стилистика — как у `/promo` и `/promothanks` (тёмная тема, фиолетовые градиенты, landing-theme.css). Поп-ап — через существующий `ResponsiveDialog` (на мобиле снизу как приложение).

## Структура страницы `/quiz`

Полноценный лендинг по образцу промо:
1. **Hero** — заголовок «WBGen — первый и лучший в мире сервис генерации эффективных карточек товаров для маркетплейсов», подзаголовок, главная кнопка «Хочу попробовать» (открывает квиз).
2. **Примеры работ** — слайдер до/после (переиспользуем `BeforeAfterSliderNew`).
3. **Как это работает** — 3-4 шага.
4. **Фичи / выгоды** — компактная сетка.
5. **Соц. доказательство** — отзывы/кейсы (короткий блок).
6. **Финальный CTA** — повторная кнопка «Хочу попробовать».

## Логика квиза

### Вопрос 1 — «Кто вы?» (single choice)
Подзаголовок на всех шагах: *«WBGen — первый и лучший в мире сервис генерации эффективных карточек товаров для маркетплейсов. Уточните, какие цели вы преследуете в нашем сервисе»*.

Варианты:
- Продавец на маркетплейсах
- Менеджер маркетплейсов
- Продавец на Avito и подобных площадках
- Дизайнер карточек
- Начинающий специалист
- Другое (→ ветка селлера)

### Вопрос 2 — «Какие цели вы преследуете?» (multi-select, зависит от вопроса 1)

**Селлер / Другое:**
- Сократить расходы на дизайнера
- Тестировать больше вариантов карточек для повышения эффективности рекламы
- Увеличить доход за счёт улучшения дизайна карточек
- Сократить штат и перекинуть задачи на менеджера
- Другое

**Менеджер маркетплейсов:**
- Оптимизировать свою работу
- Затрачивать меньше времени на дизайн карточек
- Эффективнее увеличить доход продавцу
- Дополнительно зарабатывать
- Другое

**Продавец на Avito:**
- Увеличить охват объявления за счёт фотографий
- Сделать фотосессию без фотографа
- Сократить затраты на фото
- Поднять позиции объявления в ранжировании
- Увеличить доход за счёт изменения карточек
- Другое

**Дизайнер карточек:**
- Сократить время на производство карточек в несколько раз
- Увеличить результативность карточек для селлера (продающие)
- Улучшить качество работы
- Увеличить заработок
- Другое

**Начинающий специалист:**
- Найти заказчиков
- Создавать эффективные карточки
- Освоить новую профессию
- Другое

### Шаг 3 — Видео (зависит от ветки)
- **Видео 1** → Селлеры, Менеджеры, Другое
- **Видео 2** → Avito
- **Видео 3** → Дизайнеры, Начинающие

Под видео подпись: *«Посмотрите обязательно — всего минута. Это поможет понять, нужен ли вам сервис и сможете ли вы закрыть свои задачи»*.

Кнопка: **«Хочу попробовать»** → переход на `/quiz/thanks`.

Видео встраиваются как iframe Kinescope/YouTube (ссылки добавим как константы — заменишь на реальные).

### `/quiz/thanks`
По образцу `/promo/thanks`: чек-иконка, «Спасибо, что доверились! Рады, что решили попробовать сервис. Сейчас переведём вас на регистрацию — пройдите её и следуйте инструкциям», кнопка «Перейти сейчас», автопереход через 10 сек на `/auth?tab=register`.

## Трекинг
Только финальные цели:
- `quiz_completed` — при клике «Хочу попробовать» на шаге видео
- `quiz_thanks_loaded` — на маунте `/quiz/thanks`
Отправка через Яндекс.Метрику + Meta Pixel (как сделано в `PromoThanks`).

## Технические детали

**Новые файлы:**
- `src/pages/Quiz.tsx` — лендинг
- `src/pages/QuizThanks.tsx` — страница благодарности (копия `PromoThanks` с другим текстом)
- `src/components/quiz/QuizDialog.tsx` — поп-ап-квиз (ResponsiveDialog)
- `src/components/quiz/quizConfig.ts` — конфиг веток, целей, маппинг на видео
- `src/components/quiz/QuizStepAudience.tsx`, `QuizStepGoals.tsx`, `QuizStepVideo.tsx` — шаги

**Маршруты в `src/App.tsx`:**
```
<Route path="/quiz" element={<Quiz />} />
<Route path="/quiz/thanks" element={<QuizThanks />} />
```
Lazy-load. SupportWidget скрыть (добавить `/quiz` в isHidden в `SupportWidgetWrapper`).

**Видео:** константы `VIDEO_URLS = { sellers: '', avito: '', designers: '' }` в `quizConfig.ts` — пустые плейсхолдеры, заменишь на свои Kinescope/YouTube ссылки.

**Стилизация:** `landing-theme.css`, тёмная тема `#111111`, фиолетовые градиенты, framer-motion на переходах между шагами квиза, прогресс-бар сверху поп-апа (1/3 → 2/3 → 3/3).

- Без аналитики по шагам — только финал.
- Без новых edge-функций.
