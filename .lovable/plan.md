

## Проблема

Предыдущий фикс `.toLowerCase()` был основан на неверном предположении. Документация Gemini API (подтверждено UCloud docs, LiteLLM GitHub issue) показывает, что `imageSize` принимает **верхний регистр**: `"1K"`, `"2K"`, `"4K"`.

Отправка `"2k"` (lowercase) — невалидное значение, API игнорирует его и генерирует в максимальном разрешении (~3600x4800, 4K).

Это объясняет:
- Почему после деплоя фикса Tide всё равно вышел в 4K
- Почему файлы по-прежнему большие (~2+ МБ)
- Почему генерация долгая (~2 минуты на 1 карточку) — 4K-изображение тяжелее обрабатывать

## Исправление

Одна строка в одном файле.

**Файл: `supabase/functions/process-google-task/index.ts`** (строка 121)

Убрать `.toLowerCase()` — значение из БД уже в правильном формате:

```typescript
// БЫЛО (сломано):
imageSize: imageResolution.toLowerCase(),  // "2k" — невалидно

// СТАЛО (правильно):
imageSize: imageResolution,  // "2K" — как требует API
```

После деплоя изображения будут генерироваться в реальном 2K (2048×3072), а не в 4K. Это уменьшит размер файлов вдвое и ускорит генерацию.
