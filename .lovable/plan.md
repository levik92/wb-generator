

## Лендинг для рекламного трафика + страница "Спасибо"

### Что делаем

Две изолированные страницы, не связанные с основной навигацией сайта:

1. **`/promo`** — короткий продающий лендинг с фактами, цифрами и кейсами WBGen
2. **`/promo/thanks`** — промежуточная страница "Спасибо" с авторедиректом на `/auth?tab=register`

### Структура лендинга `/promo`

Одна вертикальная страница в стиле сайта (тёмная тема `#111111`), без шапки и футера основного сайта:

1. **Hero-блок** — логотип WBGen, заголовок-оффер ("Карточки для маркетплейсов за 3 минуты"), подзаголовок с ключевой выгодой, кнопка CTA
2. **Блок цифр** — 4 ключевых метрики (например: "10 000+ карточек создано", "3 минуты на карточку", "+200% к конверсии", "от 490₽/мес")
3. **Проблема → Решение** — короткий блок "без дизайнера, без Photoshop, без ожидания"
4. **Кейсы** — переиспользуем компонент `CasesShowcase` с before/after слайдерами и цифрами роста конверсии
5. **Как это работает** — 3 шага (загрузи фото → ИИ создаёт дизайн → скачай и размести)
6. **Финальный CTA** — кнопка "Зарегистрироваться бесплатно" → ведёт на `/promo/thanks`

### Страница `/promo/thanks`

Полноэкранная страница с:
- Иконка-галочка / анимация
- Текст "Спасибо, что доверились нам!"
- Подтекст "Сейчас мы переведём вас на страницу регистрации..."
- Авторедирект через 3 секунды на `/auth?tab=register`
- Кнопка "Перейти сейчас" для нетерпеливых

### Технические детали

**Новые файлы:**
- `src/pages/Promo.tsx` — лендинг
- `src/pages/PromoThanks.tsx` — страница спасибо

**Изменения в существующих файлах:**
- `src/App.tsx` — добавить два маршрута: `/promo` и `/promo/thanks` (без `ProtectedRoute`, без `AuthRedirect`)

**Адаптивность:** мобильная и десктопная версия (используем существующие Tailwind-утилиты и паттерны из landing-theme.css)

**Изоляция:** страницы не добавляются в навигацию, sitemap, футер — доступны только по прямой ссылке для рекламного трафика

**Переиспользование:** `CasesShowcase` для блока кейсов, стили из `landing-theme.css`
