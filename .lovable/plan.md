

## Проблема

Функция `regenerate-single-card-banana` **находит** `style_description` из исходного задания (строки 214-229) и даже строит обогащённый промпт (строки 232-242), но **этот промпт нигде не используется**. Вместо этого она делегирует работу оркестратору `process-generation-tasks-banana`, который строит промпт **заново** из данных задания в БД.

При этом в `insert` нового задания (строки 156-169) **не сохраняются** поля `style_description` и `unified_styling`. Оркестратор читает задание из БД, видит:
- `job.unified_styling` = `null` → `styleAnalysisDone = true` (пропускает стилизацию)
- `job.style_description` = `null` → `styleDescription = null`

**Результат:** стиль полностью теряется при перегенерации.

## Решение

**Файл:** `supabase/functions/regenerate-single-card-banana/index.ts`

1. **Перенести поиск `style_description` до создания задания** — сначала найти стиль, потом создать job.
2. **Сохранить `style_description` в новом задании** — добавить поле в `insert` объекта job, чтобы оркестратор подхватил его автоматически.
3. **Убрать лишний код построения промпта** — промпт строится оркестратором, локальный `getPromptTemplate` не нужен.

Конкретные изменения:
- Блок поиска `styleDescription` (строки 214-229) переместить **перед** `insert` задания (до строки 156).
- В `insert` на строке 158 добавить: `style_description: styleDescription` (если найден).
- Удалить строки 231-242 (построение локального промпта, который нигде не используется).

После этого оркестратор прочитает `job.style_description` из БД и автоматически добавит стиль во все промпты через существующую логику в `getPromptTemplate` (строка 163 оркестратора).
