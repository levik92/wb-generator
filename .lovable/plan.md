## Проблема

В прошлый раз я заменил `object-cover` → `object-contain` только в развёрнутой сетке (expanded grid). Маленькие квадратные превью в строке каждой генерации (56–64px) рендерятся через отдельный компонент `HistoryAvatarImage` (History.tsx, строки 61–79), который ты ранее настраивал на «обрезанный кроп» хаком:

```tsx
className="absolute left-0 top-1/2 w-full max-w-none -translate-y-1/2 object-fill"
style={{ height: "auto", minHeight: "100%" }}
```

Этот хак растягивает фото на всю ширину контейнера, центрирует по вертикали и обрезает верх/низ — поэтому видна «увеличенная часть» изображения, а не вся карточка.

## Что поменяю

Один компонент, History.tsx, строки 61–79:

- Убрать абсолютное позиционирование и `object-fill`.
- Использовать обычный поток + `object-contain`, чтобы фото вписалось целиком в квадрат 56×56 / 64×64.
- Контейнеру-квадрату (строки ~1074, ~1093) добавить нейтральную подложку `bg-muted/40`, чтобы пустое пространство при леттербоксинге выглядело аккуратно (рамка-квадрат у иконки остаётся).

Никакой другой логики не трогаю — только классы у `HistoryAvatarImage` и подложка у двух обёрток-квадратов.

## Результат

В каждой строке Истории квадратик-превью покажет всю карточку/обложку видео целиком, без обрезки сверху/снизу. Развёрнутая сетка уже работает корректно с прошлой правки.
