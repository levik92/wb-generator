# Жалобы пользователей: «не запускается генерация» + «не открываются изображения и новости»

## Что показал анализ (фактами из БД и сервера)

### 1. [grenader99rus@mail.ru](mailto:grenader99rus@mail.ru) — генерация запускается, но Gemini блокирует контент

- 48 jobs всего, 26 fail. Все 26 — на товарах с защищёнными брендами: «Topps», «Formula 1», «Match Attax».
- В `generation_tasks.last_error` стоит `no_image_generated:PROHIBITED_CONTENT` или `unknown` (то же самое, без явного признака).
- Деньги (токены) корректно возвращаются: на каждый fail — refund 5 токенов.
- Но в UI (скриншот «Ошибка генерации / Не удалось запустить генерацию») показывается общий текст из catch-блока `GenerateCards.tsx` (строки 1134–1140), который **скрывает реальную причину**. Пользователь думает, что «не запустилось», хотя технически запуск прошёл, а Gemini отказал.

### 2. [Grimm5@list.ru](mailto:Grimm5@list.ru) — генерация работает идеально

- 23 job из 23 успешны, fail = 0. За последние 5 дней нагенерил Montale-парфюмы без проблем.
- Жалоба «при подготовке падает» — это тот же самый общий тост из catch-блока, который мог сработать в любом сетевом сбое (compress, upload, invoke). Без логов конкретную причину не видно — отлавливаем телеметрией.

### 3. «Не открываются изображения в истории» — реальная клиентская проблема

- Бакет `generated-cards` публичный, доступен (HTTP 200, TTFB 0.8 с).
- НО каждая картинка — **полный PNG ~1.7 МБ**. На странице истории `History.tsx` 20+ превью 56×56 пикселей грузят полный файл (нет `loading="lazy"`, нет ресайза, нет `srcset`).
- При 20 строках истории = 30+ МБ загрузки одновременно. На мобильном/3G — таймаут или OOM в Safari → выглядит как «не загружается».
- Бакет уже 56 ГБ, 26 582 файла — будет только хуже.

### 4. «Не открываются новости» — это часть медленной загрузки

- Таблица `news`: 40 опубликованных, RLS правильный, контент маленький (~500 символов на запись). Запрос лёгкий.
- Реальная причина: вся PWA загружается дольше из-за того же мусора в `public/` (3 файла иконок по 1.1 МБ — `apple-touch-icon.png`, `icon-192x192.png`, `icon-512x512.png`), плюс service worker их кэширует.

## Что предлагаю сделать

### A) Понятные ошибки при генерации (для grenader99rus и аналогичных случаев)

**Бэкенд** — `supabase/functions/process-generation-tasks-banana/index.ts`:

- При финализации job'а, когда все/часть тасков упали, проверять `last_error` всех тасков и заполнять `generation_jobs.error_message` человеческим текстом:
  - содержит `PROHIBITED_CONTENT` или `IMAGE_OTHER` → «ИИ заблокировал контент. Уберите названия брендов, лицензионных персонажей и торговых марок из описания и/или фото и попробуйте снова».
  - содержит `google_503_both_keys_failed` → «Серверы ИИ временно перегружены. Попробуйте через 1–2 минуты».
  - содержит `STOP` или только `unknown` → «ИИ вернул пустой ответ. Часто это связано с защищённым контентом или нечитаемым фото — попробуйте другое описание или фото».
  - иначе — текущий «Не удалось сгенерировать N из N карточек».
- То же — для `regenerate-single-card-banana/index.ts` (одиночная перегенерация).

**Фронтенд** — `src/components/dashboard/GenerateCards.tsx`:

- В `startJobPolling` (~строка 850–860): при `job.status === 'failed'` показывать тост с текстом из `job.error_message`, а не общий «Ошибка». Тост с `duration: 8000`.
- В catch-блоке `handleGenerate` (1134–1145) различать ошибки:
  - `error?.context?.status === 402` или `.message includes 'Insufficient'` → «Недостаточно токенов».
  - `error?.message includes 'upload'` → «Не удалось загрузить изображение. Проверьте интернет и попробуйте снова».
  - `FunctionsFetchError` или нет ответа → «Не удалось связаться с сервером. Проверьте интернет».
  - иначе — текущий текст + дополнительно вызвать `useSecurityLogger.logSuspiciousActivity('generation_client_error', { stage, message })`, чтобы причины Grimm5 и других стали видны в логах без скриншотов.

### B) Быстрая загрузка превью истории

**Фронтенд** — `src/components/dashboard/History.tsx`:

- Ко всем `<img>` (строки 754, 1046, 1059, 1184, 1264) добавить:
  - `loading="lazy"`
  - `decoding="async"`
  - явные `width`/`height` (предотвращает layout shift и ускоряет рендер).
- Для превью 56×56 / 64×64 (строки 1059) использовать Supabase image transform:
  ```
  ?width=128&quality=70
  ```
  В Supabase Storage есть встроенная трансформация на лету (через `?width=`/`?height=`/`?quality=` query-params для public URL). Это даст вместо 1.7 МБ ~10–15 КБ. Для grid-сетки развёрнутых карточек (строка 1184, отображение ~150–200px) — `?width=300&quality=80`.
- Проверить, что добавлять трансформации только если URL ведёт на сам Supabase (по hostname), чтобы не сломать старые ссылки или внешние.

### C) Чистка PWA-мусора (общая скорость загрузки сайта)

**Файлы** (удалить):

- `public/apple-touch-icon.png` (1.1 МБ — мусор, не используется)
- `public/icon-192x192.png` (1.1 МБ — мусор)
- `public/icon-512x512.png` (1.1 МБ — мусор)
- `public/apple-touch-icon-new.png` (70 КБ — дубликат `pwa-icon-512.png`)

`**index.html**` — заменить одну общую ссылку на правильный набор:

```html
<link rel="apple-touch-icon" sizes="180x180" href="/pwa-icon-512.png">
<link rel="icon" type="image/png" sizes="192x192" href="/pwa-icon-192.png">
<link rel="icon" type="image/png" sizes="512x512" href="/pwa-icon-512.png">
```

`**vite.config.ts**` (Workbox) — проверить, что в `globPatterns` не попадают удалённые файлы; если паттерн вида `*.png` — добавить `globIgnores` или сузить.

### D) Лоадер на странице истории (опционально, но желательно)

В `History.tsx` уже есть `loading` skeleton, но между «закончилась загрузка списка» и «прорисовались превью» сейчас белые квадраты. Добавить плейсхолдер (`bg-muted animate-pulse`) внутри `<div>` обёртки превью до момента, пока картинка не загрузится (через `onLoad` снимать класс).

## Что НЕ делаем

- Не перегенерируем существующие 56 ГБ картинок — это разовая трата на много дней. Трансформация через query-params покрывает все старые URL автоматически (это ключевое преимущество выбранного решения).
- Не блокируем ввод бренда в форме — пусть пишет, просто покажем подсказку и понятную ошибку.
- Не возвращаем токены повторно — у grenader99rus всё уже вернулось по логам refund.

## Эффект

- Пользователь grenader99rus сразу увидит «AI блокирует бренды» — вопрос в поддержку отпадёт.
- Подобные случаи у других тоже станут понятны через тосты + телеметрию.
- История генераций будет грузиться в 100–200 раз быстрее (1.7 МБ → 10–15 КБ на превью).
- Первый заход на сайт станет быстрее на ~3.3 МБ.
- На действующем кэше PWA не будет старых «жирных» иконок.
