## Что делаю

Загружено новое изображение фавикона (фиолетовый квадрат с молнией). Нужно поставить его как иконку сайта во всех местах, где раньше показывался старый/дефолтный favicon.

## Исходные файлы уже подготовлены

Из загруженного PNG созданы все нужные размеры и сохранены в `public/`:
- `favicon.ico` — для классических браузеров
- `favicon.png` (32×32) — стандартный PNG-фавикон
- `pwa-icon-192.png` — для Android home-screen
- `pwa-icon-512.png` — для splash-screen и больших превью
- `apple-touch-icon.png` (180×180) — для iOS home-screen

## Что осталось сделать в коде

### 1. `index.html` — обновить ссылки на иконки
Сейчас:
- `apple-touch-icon` ошибочно указывает на `pwa-icon-512.png` вместо отдельного `apple-touch-icon.png`
- нет отдельной ссылки на `favicon.png`
- нет version-параметра, поэтому браузеры продолжают отдавать закэшированную старую иконку

Поменяю блок на:
```html
<link rel="icon" href="/favicon.ico?v=3" sizes="any" />
<link rel="icon" type="image/png" sizes="32x32" href="/favicon.png?v=3" />
<link rel="icon" type="image/png" sizes="192x192" href="/pwa-icon-192.png?v=3" />
<link rel="icon" type="image/png" sizes="512x512" href="/pwa-icon-512.png?v=3" />
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png?v=3" />
<link rel="manifest" href="/manifest.json?v=3">
```
`?v=3` заставит браузер и CDN скачать иконку заново, а не доставать из кэша.

### 2. `public/manifest.json` — поднять версию ссылок
Внутри `icons[].src` тоже добавлю `?v=3`, иначе установленный PWA продолжит показывать старую иконку.

## Что НЕ трогаю
- Service Worker / vite-plugin-pwa — этого в проекте нет.
- Никакие другие файлы и компоненты.

## После применения

Это **фронтенд**-изменение. Чтобы новая иконка появилась на `wbgen.ru` / `wbgen.online`, нужно нажать кнопку **Publish → Update** после того, как я внесу правки.

На устройствах, где сайт уже открывали:
- Обычные браузеры подхватят новую иконку благодаря `?v=3` сразу.
- Установленные PWA на телефоне — может потребоваться удалить и установить заново (это нормально для PWA, на это никак не повлиять из кода).
