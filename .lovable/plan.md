

# Оптимизация производительности wbgen.ru

## Текущие показатели (мобильные)

| Метрика | Сейчас | Цель |
|---|---|---|
| Performance | 60 | 80+ |
| LCP (лаб.) | 13.4с | < 4с |
| FCP (лаб.) | 5.5с | < 2.5с |
| LCP (реальн.) | 4.3с | < 2.5с |

## Что будем делать

### 1. Отложить загрузку telegram-web-app.js
Сейчас скрипт Telegram блокирует рендер на ~1 секунду, хотя нужен только внутри Telegram. Добавим `defer`.

**Файл**: `index.html`

### 2. Неблокирующая загрузка шрифтов Google Fonts
Сейчас `@import` в CSS блокирует рендер на ~780мс. Уберём `@import` из CSS и добавим `<link>` с паттерном `media="print" onload="this.media='all'"`.

**Файлы**: `src/index.css`, `index.html`

### 3. Lazy-загрузка тяжёлых страниц
`Dashboard`, `Admin`, `Auth` сейчас грузятся сразу в бандл (~524 КиБ). Переведём на `React.lazy()` — они нужны только авторизованным пользователям.

**Файл**: `src/App.tsx`

### 4. Отложить регистрацию Service Worker
Добавим `injectRegister: 'script-defer'` в конфиг PWA, чтобы `registerSW.js` не блокировал рендер.

**Файл**: `vite.config.ts`

### 5. Исправить доступность (Accessibility: 81)
- Убрать `user-scalable=no` и `maximum-scale=1.0` из viewport meta — мешает зуму
- Добавить `aria-label` на кнопки без текста в хедере лендинга

**Файлы**: `index.html`, `src/components/landing/LandingHeader.tsx`

## Ожидаемый результат
